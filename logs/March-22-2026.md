# LeanTree Changes — March 22, 2026

## Summary

Partial proof tree extraction with per-subtree error isolation. LeanTree now handles three failure modes gracefully, producing expanded trees with errors annotated on specific branches rather than aborting the entire tree.

## Lean 4.19 → 4.29 Upgrade (prerequisite)

LeanTree originally required Lean 4.19. Upgraded to 4.29.0-rc6 to support `grind` and align with modern Lean toolchains.

### lean-repl patches (`leantree/lean-repl/`)
- `REPL/Lean/InfoTree.lean` — Added missing `Info` constructors: `ofDocElabInfo`, `ofDocInfo`, `ofErrorNameInfo`
- `REPL/PaperProof/BetterParser.lean` — `List.bind` → `flatMap`, `List.join` → `flatten`, `HashSet.empty` → `emptyWithCapacity`
- `REPL/Main.lean` — `updateLet!` takes extra `Bool` (nonDep) argument in 4.29

### Python-side fixes
- `repl_adapter/data_extraction.py` — `json.JSONDecoder.raw_decode()` to handle extra JSON objects in REPL stdout
- `utils.py` — Auto-apply `nest_asyncio` when running in an event loop (Jupyter support)
- `repl_adapter/ast_parser.py` — Bounds check in `skip_to_argument_end`; wrap bare `LeanASTLiteral` roots in synthetic `LeanASTObject`

## Postprocessor changes (`data_extraction/tree_postprocessor.py`)

### New: `_merge_intro_arguments()`
In Lean 4.29, the REPL splits `intro n h1` into two proof steps. This detects bare identifiers following `intro` and merges them back, with a `_TACTIC_KEYWORDS` blocklist to avoid merging real tactics like `omega`.

### New: `_fix_incomplete_using_clause()`
REPL sometimes splits `simpa [...] using <term>` leaving `simpa [...] using` without its argument. Reads the missing argument from source text and appends it.

### New: `_decompose_unresolved_nested_by()`
When a nested `by <tactic>` block has an error, the REPL doesn't report it as a separate step. This detects un-decomposed `by` blocks, replaces with `by sorry`, and creates a synthetic spawned-goal child so the tree builder replays the nested tactic separately.

### New: `_extend_spans_for_trailing_content()`
Extends sub-spans to cover trailing tactic arguments not included in the REPL's reported span (e.g., `using hpd3`), stopping before closing delimiters.

### Refactored: constructor extraction for `cases`/`induction ... with`
Three-tier fallback: AST parsing (Lean <4.29) → source text regex (Lean 4.29+) → goal tag extraction (last resort). The AST structure changed in 4.29.

### Disabled: `_remove_by_sorry_in_have()`
Stripping `:= by sorry` from `have` causes parse errors with custom notation in 4.29 (e.g., `n !`). Method is now a no-op.

## Error handling (core feature)

### Three failure modes handled

1. **Reference/syntax errors** (e.g., unknown constant `factorial_po`): REPL sorry-recovers, returns goals + error messages → error annotation on the edge, tree fully expands.

2. **Explicit `sorry`**: Detected before REPL replay → error edge created directly, no REPL interaction for that branch.

3. **Tactic failure** (e.g., `omega` can't prove `n < n`): Exception caught per-tactic → error child node as terminal leaf.

### Files changed

- **`core/proof_tree.py`**: Added `error: str | None` to `ProofTreeNode` and `ProofTreeEdge`. Added `has_error()` recursive check. Updated `is_solved()` to return False when errors present. Updated `pretty_print()` to show errors inline. Updated serialization/deserialization.

- **`data_extraction/tree_builder.py`**: Added sorry detection before replay. Wrapped `apply_tactic()` in try/except for per-tactic error isolation. Propagated `error_messages` from branches to edge annotations. Added fallback to check `branch.error_messages` when `sub_branches` is empty (sorry closed all goals but had errors).

- **`repl_adapter/interaction.py`**: Modified `_send_and_receive_async` to store errors as `_error_messages` when the REPL also returns valid goals (sorry-recovery), instead of throwing. Added `error_messages` field to `LeanProofBranch`. Propagated error messages in `apply_tactic_async`, including to the calling branch when `next_states` is empty.

- **`repl_adapter/data_extraction.py`**: Removed hard rejection of files with Lean compilation errors — lets valid compilation units through for per-tactic error handling.

- **`core/project.py`**: Reads source text and passes it through to `ProofTreePostprocessor.transform_proof_tree()`.

## Known limitations

- Wellfounded induction, `<;>` combinator, `calc`, `conv` still unsupported
- `cases`/`induction ... with` pattern decomposition uses source-text fallback in 4.29 (AST structure changed)
- `_remove_by_sorry_in_have` fully disabled (should be conditional on parse errors)
- `_merge_intro_arguments` uses keyword blocklist instead of AST-based detection
- Cross-theorem sorry propagation not tracked
